# LioraLang Project Documentation

## Что это за проект

LioraLang - offline-first приложение для изучения языков через собственные колоды, flashcards и spaced repetition.

Проект поддерживает две целевые платформы:

- `desktop`: Electron + React + SQLite
- `web`: Vite + React + IndexedDB + PWA

Обе версии используют один React-код, общие бизнес-правила и слой platform services, который прячет различия между средами выполнения.

## Для кого этот документ

Этот документ нужен разработчику, который впервые открывает проект и хочет быстро понять:

- что делает приложение;
- как оно разложено по слоям;
- где находится логика страниц, фич и доменных сущностей;
- чем отличаются web и desktop;
- какие части кода сейчас сильные, а какие требуют рефакторинга.

## Краткий вывод по проекту

### Что в проекте хорошо

- Есть сильная идея: один UI-код для двух платформ.
- Есть реально полезная platform abstraction.
- Бизнес-логика SRS и import/export вынесена в отдельные use case-ы.
- Web-версия уже использует lazy routes, IndexedDB и service worker.
- Desktop-версия уже работает через `preload` и `contextIsolation`.

### Что в проекте сейчас слабее всего

- Реальные границы FSD соблюдаются не везде.
- В проекте есть несколько крупных монолитных файлов.
- Есть дублирование desktop-адаптеров.
- Документация и README были частично устаревшими.
- Линтер на момент аудита падает.

Аудит и документация делались поверх рабочего дерева, где уже были пользовательские незакоммиченные изменения в:

- `electron/main.js`
- `src/widgets/BrowseDeckDetailsPanel/**`
- `src/widgets/BrowseDecksPanel/**`
- `packages/shared/src/lib/clipboard/**`

Поэтому документация описывает текущее состояние проекта максимально точно, но без вмешательства в эти пользовательские правки.
